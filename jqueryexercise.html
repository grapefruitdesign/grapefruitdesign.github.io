II. HTMX Question

Question:
Is it possible to have multiple trigger types in HTMX? For example, suppose that I want to trigger the request when a select element (see the element below) changes or load and then display the response in an area of my page with id="result". So, if I have the following select element, how would I use/code the htmx attributes for this scenario? Notice that the POST request is already coded.

Answer:
Yes. HTMX allows multiple trigger types, and this is done using the hx-trigger attribute with a comma-separated list of events.

Here's how you could send a POST request both when the page loads and when the user changes the selection:

<select 
  name="city"
  id="city" 
  hx-post="/api/search"
  hx-target="#result"
  hx-trigger="load, change">
    <option>San Francisco</option>
    <option>New York</option>
    <option>Las Vegas</option>
</select>

<!-- Area where the server response will be displayed -->
<div id="result"></div>

Explanation:
hx-post="/api/search"
This sends an HTTP POST request to the /api/search endpoint on your server.

  • This endpoint is where you handle the request, read the selected city from the form data, and return an appropriate response (usually HTML that HTMX will inject into the page).
  • For example, your server might look up hotels or weather data for the selected city and send back an HTML snippet.

hx-target="#result"
This tells HTMX where to place the server’s response — in this case, it will replace the contents of the element with id="result".

hx-trigger="load, change"
This specifies two trigger events:

  • load → fires once when the page finishes loading, sending an initial POST request using the default selected option.
  • change → fires whenever the user selects a different option.
